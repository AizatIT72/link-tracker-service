# LinkTracker

## LinkTracker – Telegram-бот, который отслеживает изменения на веб-страницах и оперативно информирует пользователя о них.

Система, которая решает реальную проблему разработчика: когда следишь за десятками репозиториев и технических дискуссий одновременно, то постоянно теряешь важные обновления или тратишь время на ручную проверку каждого ресурса. LinkTracker централизует это: пользователь один раз подписывается на ссылки GitHub и StackOverflow, а бот сам присылает структурированные уведомления с автором, временем и превью — когда появился новый Issue, Pull Request или ответ на вопрос.

Система построена на двух независимых микросервисах. Scrapper - это ядро системы: по расписанию опрашивает GitHub API и StackOverflow API, сравнивает полученные данные с последним известным состоянием, находит новые события и сохраняет подписки пользователей в PostgreSQL. Именно здесь находится вся бизнес-логика - парсинг ссылок, определение типа ресурса, формирование читаемого уведомления с автором, заголовком и превью описания. Bot - точка входа для пользователя: принимает команды /track, /untrack, /list через Telegram, перекидывает их в Scrapper и доставляет готовые уведомления обратно пользователю. Такое разделение позволяет масштабировать сервисы независимо — Bot можно горизонтально масштабировать под нагрузку от Telegram, не трогая Scrapper, и наоборот.

## Технический стек и реализованные решения:
- Межсервисное взаимодействие через REST и gRPC (Spring gRPC + Protobuf) — gRPC используется для высоконагруженного канала Scrapper → Bot
- Хранение данных в PostgreSQL с миграциями через Liquibase; две реализации слоя данных — чистый SQL (JdbcClient) и ORM (Spring Data JPA) с переключением через конфигурацию
- Асинхронная доставка уведомлений через Apache Kafka
- Пакетная и многопоточная обработка ссылок через ExecutorService с настройкой через конфиг
- Контейнеризация через Docker и Docker Compose
- CI/CD пайплайн в GitLab CI с этапами build, lint, test
- Интеграционные тесты с Testcontainers и WireMock для мока внешних API

## Инструкция по запуску телеграм бота

- Создать .env файл, где указать TELEGRAM_TOKEN=токен_бота
- Дальше Edit Configurations -> Выбрать BotApplication в поле Name
- Далее в Environment variables указать путь до .env файла и запустить

### Либо (без создания .env файла):

- Сразу зайти Edit Configurations -> Выбрать BotApplication в поле Name
- Далее в Environment variables указать TELEGRAM_TOKEN=токен_бота
